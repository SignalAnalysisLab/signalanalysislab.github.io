Service overview

BitBucket

Primary Git repository for the organization. This is where you as a developer primarily will permanently commit code.

Confluence

Also known as wiki. Here you will find documentation for various aspects of SAL.

Docs

Coming soon. The documentation for the code. Our Continuous Integration pipeline uses Sphinx to automatically generate this. Is also available to select external customers.

Dropbox

Used for various file storage needs which doesn't suit a wiki, the data storage, or anywhere else.

Email

Provided by Google. Usually for external communications. Use WebEx Teams for internal communication.

Jira

Project management and issue tracker. Used by managers for gaining overview of projects, used by developers to track issues with software.

Jupyterhub

Coming soon. JupyterHub is used for live demonstrations of SAL code.

Storage

Coming soon. The data storage server is used to store large datasets for analysis.

WebEx Teams

Main internal communications channel. Provides chat and conference call functionality. Morning calls are usually done here. You can also get the app.

Dig's intranet

Only relevant in Trondheim. Dig's intranet for booking rooms etc.
